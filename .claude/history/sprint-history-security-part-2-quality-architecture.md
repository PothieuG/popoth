### ✅ Fait (Sprint Refactor-Architecture — livré 2026-05-08)

- **Scope arbitré Phase 1** : 5 chantiers sur 6 du prompt source ([prompts/prompt-03-architecture.md](prompts/prompt-03-architecture.md)). Le Context local PlanningDrawer a été **skippé** (pattern Sprint Hygiène-Code) après que l'inventaire a montré que l'audit était stale : `context` est lu **une seule fois** au top de [components/dashboard/PlanningDrawer.tsx:25](components/dashboard/PlanningDrawer.tsx) pour alimenter 4 hooks, jamais re-passé aux dialogs enfants. Un Context aurait remplacé ces 3 props par un provider+hook — gain marginal, churn pour rien.
- **Chantier 1** — middleware self-call → import direct. [middleware.ts:65-74](middleware.ts) faisait un `fetch(${baseUrl}/api/monthly-recap/status?...)` synchrone bloquant en Edge runtime. Extraction de la logique métier en [lib/recap/check-status.ts](lib/recap/check-status.ts) exportant `checkRecapStatus(userId, context: 'profile' | 'group')`. Edge-safe vérifié Phase 1 : `lib/supabase-server.ts` n'importe que `@supabase/supabase-js` + `process.env`, et `validateSessionToken` (`lib/session-server.ts`) utilise `jose` + cookie parsing manuel — pas de Node stdlib. La route [app/api/monthly-recap/status/route.ts](app/api/monthly-recap/status/route.ts) devient un thin wrapper qui valide la session, parse le contexte et délègue à `checkRecapStatus`. Le middleware appelle directement la fonction. Response shape de la route préservée à l'identique (pas de breaking change). Pattern d'erreur : `RecapStatusError` avec codes `PROFILE_NOT_FOUND` / `NO_GROUP` — la route mappe `404` / `400`, le middleware avale silencieusement `NO_GROUP` (l'utilisateur sans groupe n'est pas concerné par le récap groupe). Les anciennes routes wrappent les handlers via [lib/api/with-deprecation.ts](lib/api/with-deprecation.ts) qui ajoute `response.headers.set('Deprecation', 'true')`. **2 ambiguïtés découvertes en Phase 1, préservées telles quelles (zero-risk)** : (a) `/api/finances/dashboard` (GET, POST) ≠ `/api/financial/dashboard` (GET) — handlers différents, le second devient `/api/finance/summary` pour distinguer ; (b) `/api/budgets` et `/api/finances/budgets/estimated` cohabitent sous `/api/finance/budgets` et `/api/finance/budgets/estimated` (Next.js App Router permet la cohabitation file vs folder).
